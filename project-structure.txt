شرح هيكلية مشروع أرشيف الثقة التجارية
=====================================

1. نظرة عامة:
   المشروع عبارة عن منصة أرشيفية لإدارة المنتجات، الموردين، العملاء، وقطع الغيار.
   مبني باستخدام Next.js (App Router) و React.

2. صفحات الموقع الرئيسية (المسارات):
   - / (الرئيسية): لوحة التحكم (Dashboard) تعرض ملخصات وإحصائيات سريعة.
   - /login: صفحة تسجيل الدخول وتسجيل عضوية جديدة.
   - /admin: لوحة الإدارة للمشرفين (إدارة المستخدمين، الموردين، المهام، النسخ الاحتياطي).
   - /profile: الملف الشخصي للمستخدم الحالي.
   - /products: عرض قائمة المنتجات مع الفلترة والبحث.
   - /suppliers: عرض قائمة الموردين.
   - /clients: عرض قائمة العملاء.
   - /add-product: نموذج إضافة منتج جديد.
   - /add-spare-part: نموذج إضافة قطعة غيار (مرتبطة بمنتج).
   - /add-operational-accessories: نموذج إضافة مستلزمات تشغيل.
   - /add-supplier: نموذج إضافة مورد جديد.
   - /add-customer: نموذج إضافة عميل جديد.
   - /offers/create: إنشاء عرض سعر جديد.

3. العلاقات بين الأقسام:
   - المنتجات (Products) هي الكيان المركزي.
   - قطع الغيار (Spare Parts) ترتبط بمنتج "أب" (Parent Product).
   - مستلزمات التشغيل (Operational Accessories) ترتبط بمنتج "أب".
   - الموردين (Suppliers) يرتبطون بالمنتجات (كل منتج له مورد).
   - العملاء (Clients) يرتبطون بعروض الأسعار والطلبات.
   - المستخدمين (Users) يديرون النظام ولهم صلاحيات (Admin/User).

4. وظائف الباك اند المطلوبة (Backend Requirements):
   أ. المصادقة (Authentication):
      - تسجيل الدخول (Login) والتحقق من المستخدم.
      - تسجيل عضو جديد (Register) مع رفع الصورة الشخصية.
      - إدارة الجلسات (Sessions) وحماية المسارات (Middleware).

   ب. إدارة البيانات (CRUD Operations):
      - المنتجات: إنشاء، قراءة، تحديث، حذف. (مع رفع صور وملفات PDF/CAD).
      - قطع الغيار: نفس العمليات، مع التأكد من الربط بالمنتج الأب.
      - الموردين والعملاء: إدارة بيانات الاتصال والعناوين.
      - المهام والطلبات: إنشاء مهام وتعيينها للمستخدمين.

   ج. الملفات (File Storage):
      - نظام لرفع وتخزين الصور (للمنتجات، الأعضاء).
      - تخزين ملفات PDF (الكتالوجات، الأدلة) وملفات CAD.

   د. البحث والفلترة:
      - واجهة API للبحث المتقدم في المنتجات (بالاسم، الفئة، المورد).

   هـ. لوحة الإدارة:
      - إحصائيات حية (عدد المنتجات، الموردين، إلخ).
      - إدارة طلبات العضوية (قبول/رفض).
      - نظام النسخ الاحتياطي (تصدير قاعدة البيانات).
